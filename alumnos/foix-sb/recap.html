S1 · Viernes — JavaScript en Model-Driven + Skills y Agents
Viernes 16 de abril 2026 · Día 3 de 12
Presentación + Transcript
Qué se cubrió desde tu rol
  • JavaScript en formularios: cuándo usarlo y cuándo no. Lo importante: Business Rules para lógica simple, JavaScript para lo complejo. Como técnica que no programa, lo que te aplica es saber pedirle a Claude que genere este código.
  • Los 3 eventos clave: OnLoad (al abrir el formulario), OnChange (al cambiar un campo), OnSave (al guardar). Entender esto permite describir bien qué quieres automatizar.
  • Skills y Agents de Claude Code creados en vivo: cómo instruir a Claude con reglas permanentes para que trabaje mejor en tu contexto.
  • Regla de oro del bloque: si la lógica tiene que aplicarse siempre (incluyendo desde la API o un flujo), necesitas un Plugin o Power Automate, no JavaScript.
Lo más relevante para tu rol
  • Skills y Agents permanentes: puedes crear un “perfil” de trabajo que describa tu rol, tus tareas habituales y las reglas del sistema. Claude lo lee siempre al empezar, sin que tú tengas que repetirlo cada vez.
  • Los 3 eventos del formulario (OnLoad, OnChange, OnSave): no para programarlos tú, sino para describir con precisión qué quieres que pase y cuándo. Claude entiende mejor tu petición si usas estos términos.
  • Business Rules vs JavaScript: si la lógica es simple (campo obligatorio, ocultar algo), se hace sin código. Sólo necesitas JavaScript para lo complejo — y ahí es cuando le pides a Claude que lo escriba por ti.

S2 · Lunes — Instalar Claude Code + GitHub + Tu Primer Proyecto
Lunes 21 de abril 2026 · Día 4 de 12
Presentación + Transcript
Qué se cubrió desde tu rol
  • Diferencia Claude Code vs Claude.ai: Claude Code vive en el editor, lee y modifica archivos reales, ejecuta comandos. Claude.ai es el chat en el navegador donde ya trabajabas.
  • Instalar Claude Code en VS Code paso a paso, incluyendo login con la licencia Team de LitoClean.
  • GitHub desde cero: qué es Git vs GitHub, el flujo de trabajo diario (add → commit → push), los 10 comandos esenciales.
  • Expectativas realistas: Claude no sabe de tu empresa hasta que se lo cuentas. Da contexto. Revisa siempre antes de aceptar.
Lo más relevante para tu rol
  • Dar contexto a Claude: este es el salto más importante para ti. Sin contexto Claude da respuestas genéricas; con contexto (“soy técnica de laboratorio, trabajo en ensayos piloto, necesito...”) las respuestas son adaptádas a tu situación real.
  • Los 4 comandos de Git que usas tú: pull, add, commit, push. Con esos cuatro puedes contribuir al repo del curso y a cualquier proyecto del equipo. El resto es avanzado y no lo necesitas aún.
  • Expectativas realistas: Claude no sustituye el juicio técnico — lo amplia. Con una buena descripción del problema puede hacer el 80% del trabajo que te llevaría horas aprender. Tú revisas y decides.
