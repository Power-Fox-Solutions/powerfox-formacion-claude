02

Bitácora detallada

Sesión 1 14 Abr 2026 Ecosistema

Lo mejor

Entender que Claude no es solo un chatbot: puede vivir dentro del entorno de trabajo y leer el contexto del proyecto. Ver cómo Power Platform, GitHub y Claude Code encajan como piezas de un proceso abrió la puerta a imaginar aplicaciones concretas para el área financiera.

Lo que costó

La terminología técnica: "arquitectura", "entornos", "entidades". La solución fue traducir: una entidad es una tabla de Excel; un entorno es tener un archivo de prueba y uno de producción.

Prompt guardado

Pedirle a Claude que explique conceptos técnicos usando analogías del mundo financiero o contable.

Sesión 2 15 Abr 2026 Diseño de datos

Lo mejor

El módulo de diseño de tablas fue el más cercano al trabajo diario. Una buena estructura de datos en Power Platform es básicamente diseñar bien un modelo en Excel — con la diferencia de que aquí escala a toda la empresa.

Lo que costó

Configurar VS Code y entender para qué sirve cada herramienta. La diferencia entre VS Code, Git y GitHub requirió repasar después de la sesión.

Prompt guardado

"Soy controller financiero sin conocimientos de código. Explícame [concepto técnico] usando un ejemplo de Excel, contabilidad o reporting financiero."

Sesión 3 16 Abr 2026 Skills & Agents

Lo mejor

Ver skills y agents en funcionamiento en vivo. Entender que se pueden dar instrucciones permanentes (skills) para que Claude actúe con contexto fijo sin repetirle cada vez quién eres y qué necesitas.

Lo que costó

La parte de JavaScript quedó lejos del día a día. Pero lo valioso fue ver que Claude puede generarlo si se le explica en lenguaje natural qué tiene que ocurrir.

Prompt guardado

Describir una acción de negocio en lenguaje natural (si pasa X, entonces Y) y pedir que Claude lo convierta en lógica técnica.

Sesión 4 21 Abr 2026 Claude Code

Lo mejor

Instalación de Claude Code y comprensión de la diferencia práctica con Claude.ai: Claude Code toca archivos reales, ejecuta acciones y mantiene memoria del proyecto. Analogía clara: asesor externo sin contexto vs. colaborador que conoce tus archivos.

Lo que costó

El concepto de repositorio y ramas (branches). El flujo de Git —rama, commit, merge, pull request— es una lógica diferente a las "versiones" de archivos en finanzas. Requiere práctica.

Prompt guardado

Usar Claude para que revise el estado actual de un proyecto leyendo archivos directamente, sin tener que explicarle el contexto desde cero cada vez.

Sesión 5 22 Abr 2026 Primer caso real

Lo mejor

Primera experiencia práctica real: resolver formatos condicionales en Excel describiendo el problema en lenguaje natural. Resultado: 2 minutos vs. media hora buscando en Google. Ver a Claude evaluar una aplicación técnica en vivo —arquitectura, rendimiento, permisos— en 2 minutos confirmó el salto de productividad.

Lo que costó

Calibrar cuándo tiene sentido automatizar algo con Claude Code vs. resolverlo directamente en Claude.ai o en Excel. Es un criterio que se afina con el uso.

Prompt guardado

"Tengo esta hoja de Excel con [describir estructura]. Quiero que [resultado esperado]. Explícame los pasos como si no supiera nada de fórmulas avanzadas."

03

Tus 4 prompts personalizados

Resolver problemas de Excel sin saber fórmulas

Para el día a día: fórmulas complejas, formatos condicionales, tablas dinámicas. Describe el problema y Claude resuelve.

Tengo una hoja de Excel con [describir columnas y datos]. Quiero [resultado: fórmula, formato condicional, tabla dinámica, etc.]. Explícame los pasos de forma clara, como si no supiera nada de fórmulas avanzadas.

Traducir conceptos técnicos al mundo financiero

Para decodificar jerga del curso o del equipo técnico sin perder el hilo.

Soy controller financiero sin conocimientos de código. Explícame [concepto técnico] usando una analogía de Excel, contabilidad o reporting financiero. Sé concreto y evita jerga de programación.

Analizar un dataset o informe financiero

Para extraer hallazgos en formato ejecutivo: qué pasa, por qué importa, qué investigar.

Analiza este conjunto de datos [pegar o adjuntar datos]. Identifica patrones, anomalías o tendencias relevantes desde una perspectiva financiera. Presenta los hallazgos en formato ejecutivo: - qué está pasando - por qué importa - qué haría falta investigar

Automatizar un proceso repetitivo

Para identificar qué automatizar y cómo, ordenado de menor a mayor complejidad técnica.

Este es un proceso que hago manualmente cada [semana/mes]: [describir pasos]. ¿Cómo podría automatizarlo o simplificarlo usando las herramientas disponibles (Excel, Power Platform, Claude)? Dame opciones ordenadas de menor a mayor complejidad técnica.
04

¿Qué herramienta uso en cada caso?

Claude.ai

  • Explorar o prototipar ideas
  • Resolver algo puntual sin instalar nada
  • Preguntas sin contexto de proyecto
  • Rápido, sin memoria persistente

Claude Code

  • Hay archivos reales involucrados
  • El proceso es repetitivo
  • Necesitas que recuerde el contexto del proyecto
  • Modificar o generar documentos del proyecto

Excel directo

  • El problema es pequeño y ya sabes resolverlo
  • No necesitas explicación ni automatización
  • Operaciones manuales de una vez

Flujo recomendado para ti: Prototipar en Claude.ai (rápido, sin fricción) → implementar con Claude Code cuando la tarea implique archivos reales o pasos repetitivos del proyecto.

CASO PRÁCTICO

Informe mensual de cierre con Claude Code

Dar a Claude Code acceso a los archivos de datos y pedirle que genere el borrador del informe de cierre mensual.

  1. Crear carpeta /alumnos/agustin-buonomo/demos/cierre-mensual/

  3. Escribir instrucciones.md describiendo la estructura del informe esperado
  4. Pedir a Claude Code que genere el borrador a partir de los archivos
  5. Iterar: ajustar estructura, añadir variaciones de meses anteriores

Nota sobre privacidad: este repo es público. Nunca subir datos reales de LitoClean o de clientes. Usar datos ficticios o anonimizados para el demo.
