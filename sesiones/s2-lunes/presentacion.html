Instalar Claude Code en VS Code

30 min · CLAVE

💡 ¿Qué es Claude Code?

Es Claude dentro de tu editor. No genera texto para que tú copies — lee tus archivos, ejecuta comandos, edita código, consulta la terminal, navega por tu proyecto y trabaja contigo en tiempo real.

Se instala como una extensión en VS Code o se usa desde terminal. Hoy usaremos la extensión de VS Code.

🆚 Claude Code vs Claude.ai

Claude CodeClaude.ai
Vive en tu editorVive en el navegador
Lee y modifica tu código realTe responde, tú copias/pegas
Ejecuta comandos, instala paquetesNo toca tu máquina
Ideal: planear, construir, refactorIdeal: prototipar, visualizar, brainstorm
🔑

Activar tu licencia Team y entrar

  1. Revisa tu email — debería estar la invitación a la Workspace de LitoClean en Claude.ai. Si no la encuentras, mira spam o avisa al facilitador.
  2. Acepta la invitación — crea la cuenta con el correo corporativo. Si ya tienes Claude.ai personal, Claude te ofrece asociar la workspace.
  3. NO añadas método de pago — está cubierto por la licencia Team. Si Claude te lo pide, revisa que estés en el contexto correcto de la workspace.
  4. Verifica que estás en la workspace — arriba a la izquierda debes ver "LitoClean" o el nombre del workspace del equipo.
  5. Instala la extensión en VS Code — abre VS Code, busca "Claude Code" en Extensions (Ctrl+Shift+X), instala la oficial de Anthropic.
  6. Login desde la extensión — abre el panel (icono Claude en la barra lateral), pulsa "Sign in with Claude", se abre el navegador, confirma que es tu cuenta Team.
  7. Abre una carpeta — Claude Code necesita un directorio de trabajo. File → Open Folder → elige una carpeta (mejor no vacía; tampoco tu disco entero).
  8. Lanza tu primer prompt — cualquier cosa. Prueba el P1 que viene a continuación.

⚠️

Expectativas realistas (no saltes esto)

Te pedirá permiso siempre

Antes de ejecutar comandos o modificar archivos importantes, Claude pide confirmación. No te asustes — es bueno.

No confíes ciegamente

Claude se equivoca. Lee el código antes de aceptar. Revisa cada diff. Piensa antes de hacer Yes All.

No lo uses en producción sin PR

Todo cambio generado debería pasar por una Pull Request con revisión humana. Nunca commit directo a main.

No sabe de tu negocio

Claude no conoce LitoClean, Geslab, ni REMMOT hasta que se lo cuentas. Si quieres respuestas útiles, da contexto.

No tiene acceso a tu Dataverse

Trabaja con archivos locales. Para tocar Dataverse necesitas Xrm.WebApi en el código, no a través de Claude Code directamente.

Consume "mensajes" de la licencia

Cada interacción gasta cuota. Aprovecha los prompts largos y específicos en vez de muchos cortos.

⚠️ Evita darle acceso a carpetas con secretos. Si tu proyecto tiene archivos .env con credenciales, claves API o similares, asegúrate de que están en .gitignore y revisa qué lee Claude. Mejor: crea una carpeta dedicada para cada proyecto y no abras el disco entero.

GitHub Desde Cero

45 min

🌱 Git vs GitHub — no son lo mismo

Git es el programa que vive en tu ordenador y lleva el historial de tus archivos. Guarda versiones, permite volver atrás, permite trabajar en paralelo con ramas.

GitHub es una web que guarda copias remotas de tus repositorios Git y añade colaboración: compartir, revisar, proponer cambios, fusionar.

Analogía: Git es Word con control de cambios. GitHub es SharePoint donde guardas el Word y otros pueden verlo.

🎯 ¿Por qué sí o sí?

  • Historial completo — vuelves a cualquier versión pasada
  • Backup — si explota tu PC, tu código sigue ahí
  • Trabajo en paralelo — tú en una rama, yo en otra, fusionamos
  • Revisión — alguien aprueba antes de que se incorpore
  • Portfolio profesional — tus proyectos públicos son tu CV técnico
  • CI/CD — ejecuta tests y despliega automáticamente
✏️

Crear tu cuenta de GitHub

  1. Entra en github.com y pulsa "Sign up".
  2. Usa tu correo corporativo (el mismo que Claude Team idealmente). Elige un username profesional — aparecerá en URLs públicas. Formato típico: tunombre o tunombre-litoclean.
  3. Verifica el email — revisa la bandeja y confirma.
  4. Plan Free está bien para empezar. Puedes tener repos privados gratis.
  5. Activa 2FA (autenticación en dos pasos) — Settings → Password and authentication. Usa una app como Authy o Google Authenticator. Crítico en cuentas profesionales.
  6. Configura Git local — abre terminal y ejecuta los comandos de identidad (los veremos en el siguiente bloque).
  7. (Opcional) Añade foto, bio y pin de repos una vez tengas algo que enseñar.

Ramas, Pull Requests y Merge

gitGraph commit id: "init" commit id: "docs: README" branch feature/login checkout feature/login commit id: "feat: form" commit id: "feat: validation" checkout main commit id: "fix: typo" merge feature/login tag: "PR #1 merged" commit id: "docs: update"

🌿 ¿Qué es una rama (branch)?

Una copia paralela del código donde trabajas sin afectar a main. Cuando está lista, pides que se fusione. Regla de oro: nunca trabajas directo en main.

Convención típica de nombres:

  • feature/nombre-cosa — nueva funcionalidad
  • fix/nombre-bug — corrección
  • docs/nombre — documentación
  • refactor/nombre — reestructura sin cambiar comportamiento

📬 ¿Qué es un Pull Request (PR)?

Una propuesta de fusión. Dices: "he terminado esta rama, miradla y si está bien, fusionadla a main". Alguien revisa, comenta, pide cambios o aprueba.

Un buen PR tiene:

  • Título corto siguiendo Conventional Commits
  • Descripción: qué cambia, por qué, cómo probar
  • Tamaño manejable (si son 2000 líneas, pártelo)
  • Tests si aplica
  • Al menos 1 aprobación antes de merge

Conventional Commits: feat: nueva feature · fix: bug · docs: documentación · refactor: reestructura · test: tests · chore: tareas de mantenimiento · style: formato sin cambios de lógica.

Conoce Tu Claude Code — Prompts de Introspección

20 min · EN VIVO

🧭 La idea: deja que Claude te enseñe a usarlo

En vez de memorizar manuales, le preguntamos a Claude Code cómo está configurado, qué herramientas tiene y cómo trabaja con él mejor. Se convierte en su propio tutorial.

Cada uno lanzará estos prompts en su máquina. Las respuestas serán ligeramente diferentes según cada setup — esa es la gracia. Descubren SU configuración.

🗺️ Global vs Local

Claude Code lee configuración de dos sitios:

  • Global (~/.claude/): tus preferencias como persona. Estilo de código, convenciones, skills de uso general.
  • Local (.claude/ dentro del proyecto): las reglas de este proyecto concreto. Stack, estructura, conexiones, convenciones específicas.

Cuando hay conflicto, local gana sobre global.

📝 CLAUDE.md en tres líneas

Es un archivo Markdown que Claude Code lee automáticamente al arrancar. Contiene "las instrucciones para tratar con este repositorio". Puede tener:

  • Propósito del proyecto
  • Stack y dependencias clave
  • Convenciones de código
  • Comandos habituales (tests, build, deploy)
  • Reglas de "no hagas esto"
