02

Nuestro Repositorio del Curso

50 min · CLAVE

🎯 La idea: un repo vivo del equipo

Vamos a crear juntos el repositorio público Power Fox — Formación Claude Code. Aquí vivirán todas las presentaciones del curso, las demos que hagamos en clase, una biblioteca de prompts compartida, y una carpeta por cada alumno para sus apuntes y experimentos.

Cada semana añadiremos material nuevo. Al final del curso, tendréis un recurso que podréis seguir usando, compartir con colegas, o enseñar a quien se incorpore al equipo.

📁 Estructura que vamos a crear

powerfox-formacion-claude/ ├── README.md ← índice del curso ├── CLAUDE.md ← reglas del repo ├── LICENSE ← MIT ├── .gitignore │ ├── sesiones/ │ ├── s1-viernes/ │ │ └── presentacion.html │ ├── s2-lunes/ │ └── s2-miercoles/ │ ├── demos/ ← código vivo de clase │ └── weather-report/ │ ├── prompts/ ← biblioteca │ ├── claude-code.md │ └── git-workflow.md │ ├── alumnos/ ← carpeta por persona │ ├── bladimir/ │ ├── noemi/ │ └── [tu-nombre]/ │ └── .claude/ ├── skills/ └── agents/

🔓 ¿Por qué público?

  • Lo podéis compartir con cualquier persona (colegas, candidatos, comunidad).
  • Es un portfolio colectivo del equipo LitoClean trabajando con Claude Code.
  • Practicamos el flujo real de repo OSS: nadie puede pushear directo a main, solo vía PR.
  • No contendrá información sensible de proyectos reales — solo material didáctico.

⚠️ Nada sensible. En este repo no van credenciales, URLs internas, datos de clientes, ni código propietario. Material didáctico y demos. Eso queda siempre en repos privados.

Paso 1 · Crear la Organización "Power Fox BI" (si no existe)

Los repositorios profesionales viven dentro de una organización, no en una cuenta personal. Una organización permite múltiples colaboradores, gestionar permisos por equipos, y presentar la marca correctamente. El plan Free cubre todo lo que necesitamos.

  1. Entra en github.com con tu cuenta.
  2. Arriba a la derecha, pulsa el + → New organization.
  3. Selecciona el plan Free ($0/month). Es más que suficiente.
  4. Nombre de la organización: powerfoxbi (o el que ya use el equipo).
  5. Email de contacto: correo corporativo.
  6. "This organization belongs to": A business or institution → Power Fox BI.
  7. Invita miembros después (lo veremos en el Paso 5).
  8. Skip el paso de preguntas sobre el uso. Organización creada.

💡 Si la organización ya existe, salta al Paso 2 y crea el repo dentro de ella.

Paso 2 · Crear el Repositorio Público

  1. Ve a la página de la organización github.com/powerfoxbi.
  2. Pulsa New repository (verde, arriba a la derecha).
  3. Owner: powerfoxbi. Importante — no tu cuenta personal.
  4. Repository name: powerfox-formacion-claude (o similar — en kebab-case, sin espacios).
  5. Description: "Repositorio público del curso de Claude Code impartido por Power Fox BI. Presentaciones, demos, prompts y proyectos de los alumnos."

⚠️ Proteger la rama main. Inmediatamente después de crear el repo: Settings → Branches → Add branch protection rule → Branch name pattern: main. Marca "Require a pull request before merging". Esto obliga a que nadie pueda pushear directo a main — todo pasa por PR.

Paso 5 · Invitar al Equipo como Colaboradores

El repo es público (cualquiera lo ve), pero solo los colaboradores pueden pushear ramas y abrir PRs. Este es el proceso para invitar a los 8 miembros del equipo.

🎫 Opción A · Invitación al repo concreto

  1. En el repo → Settings → Collaborators.
  2. Add people → escribe el username o email del alumno.
  3. Selecciona el rol: Write (puede crear ramas y PRs, no puede borrar repo).
  4. Add → GitHub envía email de invitación al alumno.
  5. El alumno acepta desde el email o en su panel de notificaciones.

👥 Opción B · Miembro de la organización

  1. En la organización → People → Invite member.
  2. Ingresa username o email.
  3. Selecciona rol: Member.
  4. GitHub envía invitación por email.
  5. Asigna al alumno al equipo "formacion" con permisos Write.

💡 Recomendación: usa la Opción B si vas a tener más formaciones o repos. Los miembros de la organización heredan permisos más fácil. Para algo puntual, la Opción A basta.

⚠️ Roles en GitHub. Read = solo lectura · Triage = gestionar issues sin tocar código · Write = crear ramas y PRs (lo que necesitamos) · Maintain = gestionar repo sin borrarlo · Admin = control total. Para alumnos: Write.

Demo Compartida · Weather Report en /demos/

25 min

🎯 Qué vamos a construir

Un script Node.js que entra en /demos/weather-report/ del repo compartido. Lee una lista de ciudades desde CSV, llama a la API gratuita de Open-Meteo, y genera un CSV consolidado + un reporte HTML con tablas y gráfico.

Es el patrón leer datos → llamar API → generar reporte — base de muchos scripts de consultoría.

🌤️ ¿Por qué Open-Meteo?

  • Gratuita sin API key
  • Sin registro, sin límites anormales
  • Datos reales de meteorología
  • Perfecta para practicar integración con APIs externas

open-meteo.com ↗

  1. En la rama main actualizada: crea una nueva rama feature/demo-weather-report.
  2. Dentro del repo, crea la carpeta demos/weather-report/.
  3. Lanza P5 — Claude Code presenta el plan, confirmamos, construye.
  4. Crea el archivo ciudades.csv con 5-10 ciudades (Barcelona, Madrid, Lisboa, París, etc.).
  5. Ejecuta npm install y luego npm start.
  6. Verifica que se genera el CSV consolidado y el reporte HTML.
  7. Abre el HTML en el navegador — todos lo ven.
  8. Commit, push, PR, merge en main.

06

MCPs · Conectar Claude Code a Herramientas Externas

10 min

🔌 ¿Qué es un MCP?

Model Context Protocol. Es un estándar abierto que permite a Claude Code hablar con servicios externos de forma segura: GitHub, Jira, Slack, Google Drive, bases de datos, navegadores web, sistemas de archivos remotos, etc.

Los MCPs se configuran en tu Claude Code (global o local) y amplían lo que Claude puede hacer. En vez de copiar/pegar, Claude lee o escribe directamente en el servicio.

💡 Ejemplos útiles para consultoría BI

  • GitHub MCP — crear issues, revisar PRs, leer repos
  • Filesystem MCP — leer archivos fuera del proyecto actual
  • Playwright MCP — controlar un navegador para automatizar tareas
  • Postgres MCP — consultar bases de datos directamente
  • Slack MCP — enviar mensajes, leer canales
P10 — Aprende sobre MCPs y los primeros a conectar

💡 No instales todos los MCPs de golpe. Empezad por uno (GitHub o Filesystem), domínalo, y luego añades el siguiente. Cada MCP añade superficie de permisos — conviene entender qué hace antes de enchufarlo.
