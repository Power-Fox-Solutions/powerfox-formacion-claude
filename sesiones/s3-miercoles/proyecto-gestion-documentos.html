El proyecto: Gestión de Documentos del Curso

El repo se autodocumenta y se autopersonaliza. Vosotros aportáis la materia prima — los transcripts y vuestros perfiles. Tres agents la procesan. Cada uno se va con un HTML que habla de él.

01

El pitch en 30 segundos

Idea central

Vamos a construir un sistema que hace que el repo se autodocumente y se autopersonalice.

Materia prima: los transcripts de las sesiones y vuestros perfiles. Procesadores: tres agents de Claude Code. Salidas: un README por sesión y un HTML personalizado para cada uno de vosotros.

03

Anatomía del sistema

Entradas

Materia prima

  • /sesiones/ — las presentaciones HTML.
  • /RAW/ — los transcripts en markdown.
  • /alumnos/ — vuestro perfil y bitácora.
Procesadores

Tres agents

  • session-recap-generator — resume cada sesión.
  • claude-md-auditor — revisa los CLAUDE.md.
  • personal-html-recap-generator — crea tu recap personal.
Salidas

Lo que obtienes

  • README.md de cada sesión.
  • Reporte de auditoría del CLAUDE.md.
  • Tu recap.html personal en tu carpeta.
La pieza clave: la skill formation-content-writer. Es la que impone el estilo (español, directo, sin corporativismo, listas máximo de 6 items). Los agents la siguen para que todas las salidas suenen al mismo tono.

05

Tu turno: cómo lo usas

  1. Asegúrate de estar al día con git switch main y git pull. Los transcripts y los agents ya viven en main.
  2. Abre tu carpeta personal en /alumnos/[tu-nombre]/ y revisa tu README.md. Eso es tu materia prima — rol, expectativas, bitácora.
  3. Invoca el agent escribiendo en Claude Code: "Genera mi recap personalizado, mi carpeta es /alumnos/[tu-nombre]/."
  4. Revisa el plan que te muestra el agent antes de generar nada. Si te encaja, le das OK.
  5. Abre el HTML resultante en /alumnos/[tu-nombre]/recap.html. Ahí tienes tu trayectoria del curso adaptada a tu perfil.
  6. Sube tu cambio con un PR. El facilitador lo mergea y queda en main para que puedas regenerarlo más adelante con sesiones nuevas.
Cuándo regenerarlo: tras cada sesión nueva, vuelves a invocar el agent. El recap crece contigo a lo largo del curso.
06

Prompts que vais a usar

Tres prompts. Cada uno con botón de copiar. Lo pegáis en Claude Code dentro del repo.

P1 Generar tu recap personalizado
Invoca el agent personal-html-recap-generator. Mi carpeta es /alumnos/[mi-nombre]/. Quiero que cruce mi perfil, mi bitácora, las presentaciones de las sesiones y los transcripts disponibles para generar un recap.html adaptado a mi rol. Antes de generar nada, muéstrame el plan de qué vas a leer y qué secciones tendrá el HTML, y espera mi OK.
P2 Generar el README de una sesión
Invoca el agent session-recap-generator sobre /sesiones/s3-martes/presentacion.html. Genera el README.md resumen en la misma carpeta, siguiendo el formato del agent: título, fecha, objetivos, agenda, prompts, herramientas, conceptos clave, takeaways y reto. Si algún apartado no aparece en la presentación, omítelo. No inventes información.
P3 Auditar tu CLAUDE.md personal
Invoca el agent claude-md-auditor sobre /alumnos/[mi-nombre]/CLAUDE.md. Quiero un reporte estructurado por severidad (críticos, recomendados, opcionales) con propuestas de mejora ancladas a líneas concretas. No modifiques el archivo, solo genera el reporte. Si no tengo CLAUDE.md aún, avísame y dime qué debería contener para mi carpeta personal.
Consejo: antes de pegar un prompt, asegúrate de tener Claude Code abierto en la carpeta correcta del repo. El agent necesita ver el sistema de archivos para funcionar.
07

Skills que el repo ya tiene activas

Las skills viven en .claude/skills/ y los agents las siguen automáticamente. Vosotros también podéis invocarlas en cualquier conversación.

formation-content-writer

Qué hace: impone el estilo del repo — español directo, máximo 6 items por lista, ejemplos antes que teoría, sin emojis decorativos, privacidad estricta.

Cuándo invocarla: cuando le pidas a Claude un README, una guía o un resumen y quieras que suene como el resto del repo. Le dices: "sigue la skill formation-content-writer".

session-html-guide-writer

Qué hace: genera guías HTML self-contained con paleta y tipografía cuidadas, evitando el aspecto genérico de IA.

Cuándo invocarla: si quieres documentar algo que aprendiste hoy en una guía visual para tu propia carpeta. Pídeselo a Claude y menciona la skill.

Tres agents ya disponibles: session-recap-generator, claude-md-auditor, personal-html-recap-generator. Viven en .claude/agents/ del repo. Cualquier alumno puede invocarlos.
