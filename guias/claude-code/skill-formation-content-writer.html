una skill de Claude Code que se carga al abrir este repo y fija tono, idioma, privacidad y estructura

Antes: cada vez que pedías a Claude un README de sesión, una guía o un recap, el estilo dependía del prompt del día. Ahora: al detectar que la tarea es contenido didáctico dentro de este repo, Claude carga automáticamente formation-content-writer y aplica 10 reglas estables — en español, con ejemplos antes que teoría, listas de máximo 6 items, y el cuidado de privacidad del repo público.

Las 10 reglas aplicadas

Lo que Claude fuerza sin que se lo pidas

Idioma y tono Estructura Visual Privacidad
01 · Idioma
Español siempre
Títulos, ejemplos, comentarios. Términos técnicos en inglés ok; la explicación en español.
02 · Tono
Directo, profesional
Sin corporativismo vacío ni aperturas tipo "En el dinámico mundo de la IA…".
03 · Orden
Ejemplo antes que teoría
Siempre abre con el caso concreto. La abstracción va después, si hace falta.
04 · Listas
Máximo 6 items
Si hay más: se agrupan, se convierten en tabla, o se reparten en sub-listas.
05 · Código
Con lenguaje especificado
Bloques triple-backtick con bash, python, etc. Comentarios solo cuando lo obvio no basta.
06 · Markdown
H1 único, jerarquía limpia
Un solo H1 por documento. H2/H3 con criterio, no para decorar.
07 · Diagramas
Mermaid cuando ayude
Un flowchart comunica mejor que párrafos si el contenido es un proceso. Si no, texto plano.
08 · Tablas
3+ elementos, 2+ atributos
Criterio objetivo: por debajo de ese umbral, un párrafo o lista es más legible.
09 · Énfasis
Negrita para conceptos
Solo conceptos clave, nunca decoración. Los emojis, como mucho uno o dos por sección.
10 · Privacidad
Placeholders siempre
Nada de datos reales: Acme Corp, ana@example.com, GUIDs de ceros. Repo público, asumir audiencia global.

Cómo cambian las respuestas en peticiones típicas

Petición del usuario Qué hace Claude con la skill activa
"Hazme un README de la sesión 3" Abre con un ejemplo de la sesión (no con introducción), usa tabla si hay objetivos comparables, cierra con el checklist de privacidad.
"Escríbeme una explicación de git worktrees para alumnos" Primero un caso real ("quieres probar dos ramas en paralelo"), luego comandos con bash, máximo 6 pasos. Si son más, los agrupa.
"Dame un recap de la última demo" Sin frases vacías de apertura. Tabla con "qué pasó · para qué sirve · qué replicar". Emojis descartados salvo uno o dos si guían la lectura.
"Pon como ejemplo un cliente real que use Dataverse" Sustituye por Acme Corp o Contoso aunque el usuario mencione uno real. Avisa del cambio en el texto o en comentario.
"Escríbeme algo largo sobre CLAUDE.md" Descompone en secciones con H2. Ningún párrafo pasa de ~80 palabras sin un descanso visual (lista, código, diagrama, tabla).
Detalles prácticos

Lo que conviene saber sobre el alcance de la skill

Solo aplica en este repo
Es una skill de proyecto, no de usuario. Vive en .claude/skills/formation-content-writer.md y solo existe aquí. Si abres otro repo de Power Fox BI o cualquier otro proyecto, Claude escribirá con su estilo por defecto. Para replicarla en otro lado, copia el archivo.
No afecta a commits ni a código
La skill se activa cuando detecta contenido didáctico: README, guía, recap, sesión, prompt, explicación. En tareas de código (refactor, test, commit message, debug) usa su comportamiento por defecto. Los mensajes de Conventional Commits no dependen de esta skill — eso lo gobierna CLAUDE.md y commitlint.
Cómo desactivarla puntualmente
Si quieres que Claude escriba de otro modo en un turno concreto (por ejemplo para un experimento de tono), basta con pedirlo explícitamente: "ignora la skill formation-content-writer esta vez". En futuras peticiones volverá a cargarse.
Cómo invocarla por su nombre
Si escribes /formation-content-writer en Claude Code, la skill se aplica de forma explícita al siguiente prompt — útil cuando quieres convertir un texto externo al estilo del repo sin estar en modo "crear contenido nuevo".
